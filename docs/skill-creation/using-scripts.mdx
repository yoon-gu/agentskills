스킬은 셸 명령을 직접 실행하도록 지시할 수 있고, `scripts/` 디렉터리에 재사용 가능한 스크립트를 포함할 수 있습니다. 이 가이드는 단발 명령, 자체 의존성을 가진 스크립트, 에이전트 친화적 인터페이스 설계를 다룹니다.

## 단발 명령(one-off commands)

기존 패키지가 바로 해결해 준다면 `scripts/` 없이 `SKILL.md`에 명령을 직접 적어도 됩니다. 여러 생태계가 런타임 의존성 자동 해결 도구를 제공합니다.

<Tabs sync={false}>
  <Tab title="uvx">
    [uvx](https://docs.astral.sh/uv/guides/tools/)는 격리 환경에서 Python 패키지를 실행하며 강한 캐시를 사용합니다.

    ```bash
    uvx ruff@0.8.0 check .
    uvx black@24.10.0 .
    ```

    - Python 기본 포함 아님(별도 설치 필요)
    - 반복 실행이 매우 빠름
  </Tab>
  <Tab title="pipx">
    [pipx](https://pipx.pypa.io/)는 격리 환경에서 Python 패키지를 실행합니다.

    ```bash
    pipx run 'black==24.10.0' .
    pipx run 'ruff==0.8.0' check .
    ```

    - Python 기본 포함 아님
    - 성숙한 대안, OS 패키지 매니저 가용성이 넓음
  </Tab>
  <Tab title="npx">
    [npx](https://docs.npmjs.com/cli/commands/npx)는 npm 패키지를 필요 시 다운로드/실행합니다.

    ```bash
    npx eslint@9 --fix .
    npx create-vite@6 my-app
    ```

    - Node.js 환경에서 바로 사용 가능
    - 버전 고정 권장
  </Tab>
  <Tab title="bunx">
    [bunx](https://bun.sh/docs/cli/bunx)는 Bun 환경의 `npx` 대응 도구입니다.

    ```bash
    bunx eslint@9 --fix .
    bunx create-vite@6 my-app
    ```
  </Tab>
  <Tab title="deno run">
    [deno run](https://docs.deno.com/runtime/reference/cli/run/)은 URL/specifier에서 직접 실행합니다.

    ```bash
    deno run npm:create-vite@6 my-app
    deno run --allow-read npm:eslint@9 -- --fix .
    ```

    - 권한 플래그 필요
    - `--`로 Deno 플래그와 대상 툴 플래그 분리
  </Tab>
  <Tab title="go run">
    [go run](https://pkg.go.dev/cmd/go#hdr-Compile_and_run_Go_program)은 Go 패키지를 직접 컴파일/실행합니다.

    ```bash
    go run golang.org/x/tools/cmd/goimports@v0.28.0 .
    go run github.com/golangci/golangci-lint/cmd/golangci-lint@v1.62.0 run
    ```
  </Tab>
</Tabs>

단발 명령 작성 팁:

- 재현성을 위해 버전 고정
- 환경 전제(예: Node.js 18+)를 `SKILL.md`에 명시
- 명령이 복잡해지면 `scripts/`로 이동

## `SKILL.md`에서 스크립트 참조

스킬 디렉터리 루트 기준 **상대 경로**를 사용하세요.

```markdown SKILL.md
## Available scripts
- **`scripts/validate.sh`** — 설정 파일 검증
- **`scripts/process.py`** — 입력 데이터 처리
```

````markdown SKILL.md
## Workflow
1. 검증:
   ```bash
   bash scripts/validate.sh "$INPUT_FILE"
   ```
2. 처리:
   ```bash
   python3 scripts/process.py --input results.json
   ```
````

<Note>
`references/*.md` 같은 지원 문서에서도 같은 상대 경로 규칙을 적용하세요.
</Note>

## 자체 완결(self-contained) 스크립트

재사용 로직은 `scripts/`에 넣고, 의존성을 스크립트 내부에 선언하면 실행이 단순해집니다.

<Tabs sync={false}>
  <Tab title="Python">
    [PEP 723](https://peps.python.org/pep-0723/)으로 스크립트 내부 TOML 메타데이터에 의존성을 선언할 수 있습니다.

    ```python scripts/extract.py
    # /// script
    # dependencies = [
    #   "beautifulsoup4",
    # ]
    # ///
    from bs4 import BeautifulSoup
    ```

    ```bash
    uv run scripts/extract.py
    ```

    - PEP 508 버전 제약 사용 가능
    - `requires-python` 지정 가능
    - `uv lock --script`로 재현성 강화
  </Tab>
  <Tab title="Deno">
    `npm:`/`jsr:` import를 활용해 스크립트 단위로 의존성 관리가 가능합니다.

    ```typescript scripts/extract.ts
    #!/usr/bin/env -S deno run
    import * as cheerio from "npm:cheerio@1.0.0";
    ```

    ```bash
    deno run scripts/extract.ts
    ```
  </Tab>
  <Tab title="Bun">
    `node_modules`가 없으면 런타임 자동 설치를 활용할 수 있습니다.

    ```typescript scripts/extract.ts
    #!/usr/bin/env bun
    import * as cheerio from "cheerio@1.0.0";
    ```

    ```bash
    bun run scripts/extract.ts
    ```
  </Tab>
  <Tab title="Ruby">
    `bundler/inline`으로 스크립트 내부에 gem 의존성을 선언할 수 있습니다.

    ```ruby scripts/extract.rb
    require 'bundler/inline'
    gemfile do
      source 'https://rubygems.org'
      gem 'nokogiri'
    end
    ```

    ```bash
    ruby scripts/extract.rb
    ```
  </Tab>
</Tabs>

## 에이전트 친화적 스크립트 설계

에이전트는 stdout/stderr를 읽어 다음 행동을 결정합니다. 아래 원칙이 특히 중요합니다.

### 대화형 입력 금지

에이전트 실행은 비대화형 셸입니다. TTY 프롬프트/확인창/비밀번호 입력 대기는 멈춤(hang)을 유발합니다.

```
# Bad
$ python scripts/deploy.py
Target environment: _

# Good
$ python scripts/deploy.py
Error: --env is required. Options: development, staging, production.
Usage: python scripts/deploy.py --env staging --tag v1.2.3
```

### `--help`를 충실히 제공

`--help`는 에이전트가 인터페이스를 학습하는 1차 정보입니다. 옵션, 기본값, 예시를 간결히 포함하세요.

```
Usage: scripts/process.py [OPTIONS] INPUT_FILE
Options:
  --format FORMAT
  --output FILE
  --verbose
Examples:
  scripts/process.py data.csv
```

### 도움이 되는 오류 메시지

단순 "invalid input" 대신, 무엇이 잘못됐고 어떤 값을 기대하는지 알려주세요.

```
Error: --format must be one of: json, csv, table.
       Received: "xml"
```

### 구조화 출력 사용

자유 텍스트보다 JSON/CSV/TSV 같은 구조화 포맷이 파이프라인 결합성과 자동 처리가 좋습니다.

```
# Good
{"name": "my-service", "status": "running", "created": "2025-01-15"}
```

데이터는 stdout, 진행 로그/경고/진단은 stderr로 분리하세요.

### 추가 권장사항

- **멱등성**: 재시도에 안전하게 설계
- **입력 제약**: 모호 입력은 추측하지 말고 명확히 거절
- **dry-run**: 파괴적 작업에 `--dry-run` 지원
- **의미 있는 종료 코드**: 실패 유형별 코드 분리 및 `--help` 문서화
- **안전한 기본값**: 파괴적 작업은 `--confirm`/`--force` 등 보호 장치
- **출력 크기 제어**: 요약/limit/offset/출력 파일 옵션으로 과대 출력 잘림 방지
