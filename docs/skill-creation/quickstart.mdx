이 튜토리얼에서는 에이전트가 진짜 난수로 주사위를 굴릴 수 있게 해주는 스킬을 만듭니다.

## 사전 준비

- [VS Code](https://code.visualstudio.com/) with [GitHub Copilot](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot)

<Note>
이 튜토리얼은 VS Code를 사용하지만, Agent Skills는 개방형 형식입니다. 같은 스킬은 Claude Code와 OpenAI Codex를 포함한 모든 호환 에이전트에서 동작합니다.
</Note>

## 스킬 만들기

스킬은 `SKILL.md` 파일을 포함한 폴더입니다. VS Code는 기본적으로 `.agents/skills/`에서 스킬을 찾습니다. 프로젝트에 `.agents/skills/roll-dice/SKILL.md`를 생성하세요.

````markdown .agents/skills/roll-dice/SKILL.md
---
name: roll-dice
description: Roll dice with true randomness. Use when asked to roll a die (d6, d20, etc.), roll dice, or generate a random dice roll.
---

주사위를 굴릴 때는 1부터 지정한 면 수까지의 난수를 생성하는 아래 명령을 사용하세요.

```bash
shuf -i 1-<sides> -n 1
```

```powershell
Get-Random -Minimum 1 -Maximum (<sides> + 1)
```

`<sides>`를 주사위 면 수로 바꾸세요(예: 일반 주사위는 6, d20은 20).
````

이게 전부입니다. 파일 하나, 20줄 이내면 충분합니다. 각 부분의 역할은 다음과 같습니다.

- **`name`** — 스킬의 짧은 식별자입니다. 폴더 이름과 일치해야 합니다.
- **`description`** — 에이전트에게 이 스킬을 언제 써야 하는지 알려줍니다. 에이전트는 이 정보를 바탕으로 활성화 여부를 결정합니다.
- **본문** — 스킬이 활성화되었을 때 에이전트가 따르는 지침입니다. 여기서는 사용자 요청에서 면 수를 추출해 터미널 명령으로 난수를 생성하도록 안내합니다.

## 실행해 보기

1. VS Code에서 프로젝트를 엽니다.
2. Copilot Chat 패널을 엽니다.
3. 채팅 패널 하단 모드 드롭다운에서 **Agent** 모드를 선택합니다.
4. `/skills`를 입력해 목록에 `roll-dice`가 보이는지 확인합니다. 보이지 않으면 프로젝트 루트 기준 `.agents/skills/roll-dice/SKILL.md` 경로가 맞는지 확인하세요.
5. **"Roll a d20"**라고 요청합니다.

에이전트가 `roll-dice` 스킬을 활성화해야 합니다. 터미널 명령 실행 권한을 요청할 수 있으며, 허용하면 1~20 사이 난수를 반환합니다.

<Note>
도구 사용 신뢰도는 모델마다 다릅니다. 어떤 모델은 스킬 지침을 일관되게 따르지만, 어떤 모델은 스스로 답변하려고 할 수 있습니다. 터미널 명령 없이 답하면 모델 드롭다운에서 다른 모델을 선택해 보세요.
</Note>

## 동작 방식

내부에서는 다음 순서로 동작합니다.

1. **탐색(Discovery)** — 채팅 세션 시작 시 에이전트가 기본 스킬 디렉터리를 스캔해 스킬을 찾고, `name`과 `description`만 읽어 관련 가능성을 판단합니다.

2. **활성화(Activation)** — 주사위 굴리기를 요청하면 에이전트가 질문을 스킬 설명과 매칭해 `SKILL.md` 본문 전체를 컨텍스트에 로드합니다.

3. **실행(Execution)** — 에이전트가 본문 지침을 따라 요청된 면 수에 맞게 터미널 명령을 조정해 실행합니다.

이 과정은 **점진적 공개(progressive disclosure)**를 사용해, 모든 스킬 지침을 미리 로드하지 않고도 많은 스킬에 접근할 수 있게 합니다. 자세한 내용은 [스킬은 어떻게 동작하나요](/what-are-skills#스킬은-어떻게-동작하나요)를 참고하세요.
