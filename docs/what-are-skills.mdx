---

스킬의 핵심은 `SKILL.md` 파일을 포함한 하나의 폴더입니다. 이 파일에는 메타데이터(최소한 `name`과 `description`)와 에이전트가 특정 작업을 수행하는 방법을 알려주는 지침이 들어 있습니다. 스킬에는 스크립트, 템플릿, 참고 자료도 함께 담을 수 있습니다.

```directory
my-skill/
├── SKILL.md          # 필수: 지침 + 메타데이터
├── scripts/          # 선택 사항: 실행 가능한 코드
├── references/       # 선택 사항: 문서
└── assets/           # 선택 사항: 템플릿, 리소스
```

## 스킬은 어떻게 동작하나요

스킬은 컨텍스트를 효율적으로 관리하기 위해 **점진적 공개(progressive disclosure)** 방식을 사용합니다.

1. **탐색(Discovery)**: 시작 시 에이전트는 사용 가능한 각 스킬의 이름과 설명만 읽습니다. 언제 관련이 있을지 판단하는 데 필요한 최소한의 정보만 먼저 로드합니다.
     
2. **활성화(Activation)**: 작업이 어떤 스킬의 설명과 일치하면, 에이전트는 전체 `SKILL.md` 지침을 컨텍스트에 읽어들입니다.
     
3. **실행(Execution)**: 에이전트는 지침을 따르며, 필요에 따라 참조된 파일을 추가로 읽거나 함께 제공된 코드를 실행합니다.

이 접근 방식 덕분에 에이전트는 빠른 상태를 유지하면서도 필요할 때 더 많은 컨텍스트를 활용할 수 있습니다.

## SKILL.md 파일

모든 스킬은 YAML 프론트매터와 Markdown 지침이 담긴 `SKILL.md` 파일에서 시작합니다.

```mdx
---
name: pdf-processing
description: PDF 텍스트 추출, 폼 작성, 파일 병합. PDF를 다룰 때 사용합니다.
---

# PDF Processing

## When to use this skill
Use this skill when the user needs to work with PDF files...

## How to extract text
1. Use pdfplumber for text extraction...

## How to fill forms
...
```

`SKILL.md` 상단 프론트매터에는 다음 항목이 반드시 있어야 합니다.

- `name`: 짧은 식별자
- `description`: 이 스킬을 언제 사용해야 하는지에 대한 설명

Markdown 본문에는 실제 지침이 들어가며, 구조나 내용에는 특별한 제한이 없습니다.

이 단순한 형식에는 몇 가지 중요한 장점이 있습니다.

- **자기 설명적(Self-documenting)**: 스킬 작성자나 사용자는 `SKILL.md`를 읽는 것만으로도 스킬이 무엇을 하는지 이해할 수 있어, 감사(audit)와 개선이 쉽습니다.

- **확장 가능(Extensible)**: 스킬은 단순한 텍스트 지침부터 실행 가능한 코드, 에셋, 템플릿까지 다양한 복잡도를 가질 수 있습니다.

- **이식 가능(Portable)**: 스킬은 단순한 파일 집합이므로 수정, 버전 관리, 공유가 쉽습니다.
